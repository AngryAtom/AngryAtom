# AngryAtom Portfolio Roadmap

This roadmap describes how the AngryAtom GitHub portfolio should grow over time.

The portfolio exists to demonstrate engineering quality through real systems, practical documentation, security discipline, automation, operations, and continuous improvement.

## Phase 1: Foundation

Goal: make the current public portfolio immediately understandable.

- Strengthen the profile README.
- Present Tempest Lab Systems as the flagship reference architecture.
- Establish the AngryAtom Repository Standard.
- Keep current documentation depth intact.
- Improve cross-linking between profile, flagship, guides, and case studies.

## Phase 2: Tempest Flagship Maturity

Goal: make Tempest the central proof of systems engineering capability.

- Expand architecture documentation.
- Add more operational runbooks.
- Add architecture decision records.
- Expand the service catalog model.
- Add security review checklists and remediation tracking.
- Improve roadmap and status signals.

## Phase 3: Project Families

Goal: let future repositories grow without fragmenting the portfolio.

Candidate public project families:

- Tempest infrastructure components.
- Shuna, Ciel, Ranga, Benimaru, Soei, and other approved lab systems.
- SpiderFoot and OSINT/SOC lab documentation.
- Infrastructure tooling.
- PowerShell utilities.
- Security tooling.
- AI and LLM projects.
- Engineering journal.

Each new repository should follow the AngryAtom Repository Standard from the start.

## Phase 4: Maintainer Workflow

Goal: make the portfolio itself show operational maturity.

- Issue templates.
- Pull request template.
- Changelog standards.
- Documentation checks.
- Link checks.
- Dependency update process where code exists.
- Release notes for versioned tools.

## Phase 5: Public Engineering Presence

Goal: connect GitHub work to broader professional outcomes.

- Technical blog.
- Public case study index.
- Project landing pages for mature products.
- Diagrams and screenshots that clarify real systems.
- Professional contact path.

## Current Standard

Every repository should reinforce this message:

AngryAtom builds documented infrastructure, automation, security platforms, AI systems, and enterprise-grade technology projects.
